# Curate systems by contribution mechanism

The repository will classify works by the mechanism of their main contribution, not by whether they improve task success or model capability. A work belongs in the core list only when its primary contribution is a reusable runtime, execution path, serving or scheduling mechanism, training or data substrate, deployment design, neural-simulation environment, evaluation harness, reliability mechanism, or operational insight for embodied foundation models.

Model-system co-design is eligible when the system bottleneck, mechanism, implementation, and systems evidence are explicit; model architecture, loss, pruning, or training changes alone are not. Core entries should be S2 or S3, while S0 and S1 works remain outside the public catalog regardless of paper influence or code availability.
