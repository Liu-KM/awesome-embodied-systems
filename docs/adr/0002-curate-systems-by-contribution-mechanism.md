# Curate systems by contribution mechanism

The repository will classify embodied works by the mechanism of their main contribution, not by whether they improve task success or agent intelligence. A work belongs in the core list only when its primary contribution is a system abstraction, infrastructure component, execution path, resource-management strategy, deployment design, observability mechanism, safety mechanism, evaluation harness, or operational insight; model-centric works are excluded unless they contain substantial system-level design beyond ordinary inference or training code.

Core README entries should be S2 or S3: embodied infrastructure/tooling systems or strong systems contributions. S0 and S1 works may be useful background, but they should not be listed in the main README merely because they are influential embodied AI papers, open-source models, or model repositories.
